비밀번호 찾기 (Back end 부분)
- JAVA MAIL API (SMTP) 사용
- SPRING servlet-context.xml에 JAVA Bean 등록
- AJAX 사용
가입 된 이메일 입력
1. 사용자가 input 태그에 입력된 이메일을 작성하고 제출 버튼을 누르면 DB에 값이 존재하는지 확인하기 위해 JS AJAX로 GET 요청을 보냅니다.
2. GET 요청을 받은 Controller에서 MyBatis를 이용하여 DB에 있는 Users 테이블 SELECT를 진행하여 필드를 확인합니다.
3. return 값을 SELECT 결과로 지정해 Controller에서 받은 Response 값이 존재하지 않는 이메일이라면 JS에서 alert 안내창을 띄웁니다.
4. DB에 존재하는 값이라면 SMTP를 이용해 이메일 발송을 합니다.
입력 된 이메일로 임시 비밀번호 발송
1. 사용자에게 이메일을 발송하기 위해 먼저 JavaMailSender를 사용할 수 있도록 Maven에 의존성 주입을 합니다.
2. SMTP을 이용하기 위해 servlet-context.xml에 JAVA Bean 객체를 등록합니다.
3. 이때, 관리자 측 이메일은 Google 계정으로 사용합니다.
4. 보낸이를 설정하는 값에서 관리자가 실제 사용하는 PASSWORD를 입력하면 에러가 발생하기 때문에 Google 설정에서 앱 비밀번호 16자리를 획득한 값을 servlet-context.xml에 등록합니다.
5. 사용자에게 보낸 임시 비밀번호 값으로 DB Users 테이블에 UPDATE를 진행합니다.

게임 상세 정보
- AJAX 사용
- 게임 별점 부여
- 게임 리뷰 작성 가능
- 게임 이미지 슬라이드
게임 상세 페이지 상단
1. 게임에 대한 리소스들을 슬라이드 형식으로 넘길 수 있게 JS로 구현했습니다.
2. 모든 게임에 대한 정보들을 DB에 저장해놓고 Controller에서 Model 객체를 이용하여 값들을 싣습니다.
3. 게임이 할인 중이라면 정상가와 판매가를 같이 보여줍니다.
4. 게임 심의 등급 데이터를 가져와 등급에 맞는 정보들을 뿌려줍니다.
5. 사용자가 이미 구매한 게임이라면, 지금 구매/장바구니에 담기 버튼이 사라지고 이미 구매한 상품안내 버튼으로 바뀌며 장바구니에 담긴 상품이라면 alert 안내창을 띄우며 막습니다. (반복적으로 무료 게임 키 발급 악용 차단)
6. 상품을 구매한 사용자들의 리뷰 별점을 토대로 평균을 내어 게임 평점 값에 등록합니다.
게임 상세 페이지 하단 - 리뷰 작성
1. input Type text 태그 안에 리뷰를 작성 할 수 있으며 JS로 input value 값을 받아 trim()으로 공백을 지우고 값이 유효한지 검사를 합니다.
2. 게임에 대한 별점은 input 태그에 직접 작성 할 수 있지만, input range를 이용해 별점 드래그도 가능합니다.
게임 상세 페이지 하단 - 리뷰 보기 및 추천/비추천
1. 게임 리뷰가 등록이 되었다면 MY 리뷰와 최신 리뷰에 날짜순으로 보입니다.
2. 리뷰에 추천/비추천 버튼을 누르면 AJAX POST 요청을 보내 Controller에서 DB에 추천/비추천 기록을 INSERT합니다.
3. INSERT가 성공적으로 완료되었다면, response 값을 받은 JS에서 추천/비추천 버튼 색을 변경하고 COUNT를 1 늘립니다.
4. 이미 누른 추천/비추천을 다시 한 번 누르면 AJAX 요청으로 INSERT된 기록을 DELETE하고 버튼의 색, COUNT 변경을 진행합니다.
5. 추천을 누른 상태로 비추천을 누르거나 그 반대의 경우 alert 안내창을 띄워 데이터 꼬임을 방지합니다.

공지사항
- AJAX GET 요청
- 원하는 공지사항 년/월 선택 가능
- 공지사항 조회 / 추가 / 수정 / 삭제
- input file 태그 사용
사용자 - 공지사항 목록
1. 공지사항 페이지로 진입하게 되면 Default는 전체 공지사항을 보여줍니다.
2. 사용자가 원하는 년도와 월을 선택하게 될 경우 년도와 월 데이터를 AJAX에 싣고 GET 요청을 보내 기존에 있던 요소들을 REMOVE 한 뒤 DB에서 정보를 SELECT하고 Response로 반환된 데이터를 뿌려줍니다.
3. 만약 사용자가 월을 선택하지 않을 경우 해당 년도의 전체 데이터를 SELECT해서 반환합니다.
관리자 - 공지사항 조회
1. 관리자 계정으로 로그인 후 공지사항 관리 버튼을 누르면 공지사항 조회를 할 수 있습니다.
2. Pagingnation 처리를 통해 10개씩 최신순으로 보여집니다.
3. Controller에서 공지사항 페이지로 진입하기 전 DB에서 ORDER BY write_date DESC로 10개씩 SELECT를 진행 후 Model 객체에 담아 JSP로 이동합니다.
4. 공지사항에 대한 번호, 제목, 등록 날짜, 작성자를 grid 형태로 보여줍니다.
5. 원하는 공지사항 요소를 클릭하면 공지사항에 대한 detail 페이지로 진입합니다.
관리자 - 공지사항 추가
1. 공지사항 조회 목록에서 오른쪽 상단 추가 버튼을 누르면 새 글을 추가할 수 있는 페이지로 진입합니다.
2. 공지사항에 대한 정보들을 input value에 입력하게 되면 JS에서 trim()으로 공백을 지운 후 유효한 값인지 검사합니다.
3. 값이 유효하다면 오른쪽 하단에 있는 확인 버튼의 disabled 설정이 자동으로 해제됩니다.
4. input Type file을 통해 관리자가 원하는 이미지 파일을 첨부할 수 있습니다.
5. 작성이 완료되어 확인 버튼을 누르면 controller에서 DB에 INSERT를 진행합니다.
6. 이때, 이미지 파일은 MultipartFile 인터페이스와 application.propertices 파일을 이용해 프로젝트 내부에 지정된 경로로 이미지 파일을 저장합니다.
7. DB에는 파일명과 확장자 값만 저장합니다.
관리자 - 공지사항 수정
1. 원하는 공지사항을 클릭 후 detail 페이지로 이동하여 오른쪽 상단의 수정 버튼을 누르면 해당 공지사항을 수정할 수 있는 페이지로 진입합니다.
2. 공지사항 수정 페이지에서도 추가 페이지와 동일하게 input 값이 비어있는지 검사 후 확인 버튼의 disabled 설정이 자동으로 해제됩니다.
3. 첨부 파일 기능도 이하 동일합니다.
4. 확인 버튼을 클릭하면 controller로 요청을 보내 DB에 해당 공지사항의 PK를 확인 후 UPDATE를 진행합니다.
관리자 - 공지사항 삭제
1. 원하는 공지사항을 클릭 후 detail 페이지로 이동하여 오른쪽 상단의 삭제 버튼을 누르면 해당 공지사항 삭제 안내 modal 창이 뜹니다.
2. modal 창의 경우 body의 자식 요소로 position: fixed 설정을 통해 유저가 스크롤을 움직여도 항상 중앙에 위치하도록 하며, z-index를 1로 설정해 모든 요소 위에 위치하도록 하여 뒷 요소를 클릭하지 못하게 막습니다.
3. modal 창에서 삭제 버튼을 누르게 되면 controller로 요청을 보내 DB에 해당 공지사항의 PK를 확인 후 DELETE를 진행합니다.
4. modal 창에서 취소 버튼을 누르게 되면 해당 modal 창의 display가 none으로 설정됩니다.

장바구니
- 장바구니 목록을 기반으로 관련된 게임 추천 알고리즘
- 목록이 비어있다면 최신 게임 추천
- 게임 삭제
- 장바구니 정보 요약 기능
장바구니 - 게임이 비어있을 때
1. 사용자가 header에 존재하는 장바구니 버튼을 클릭하면 controller에서 사용자의 cart 테이블 정보를 SELECT합니다.
2. 정보가 없다면 Model 객체는 null로 처리가 되며 JSTL c:choose를 이용하여 null일 때는 비어있다는 문구를 보여줍니다.
3. 사용자의 게임 구매를 유도할 수 있도록 하단에 최신 게임을 추천하는 기능을 구현합니다.
4. 최신 게임 추천은 새로고침을 하면 랜덤으로 상시 변경됩니다.
장바구니 - 게임이 존재할 때
1. 장바구니에 게임이 담겨있다면 해당 게임의 정보를 가져와 표시해줍니다.
2. 해당 게임의 장르를 기반으로 하단에 같은 장르의 게임을 랜덤으로 추천해주는 알고리즘을 구현합니다.
3. 만약, 게임이 여러개 담겨있다면 여러개의 게임 중 하나의 게임의 장르를 랜덤으로 골라 DB에서 LIKE로 같은 장르의 게임을 랜덤으로 추첨하여 하단부에 뿌려줍니다.
4. 장바구니 목록에서 해당 게임을 삭제하고 싶다면 제거 버튼을 누릅니다.
5. 제거 버튼을 누르게 될 경우 controller로 요청을 보내 사용자의 cart content 컬럼에서 해당 게임의 ID를 삭제하는 DELETE를 실행합니다.
장바구니 정보 요약
1. 장바구니에 존재하는 게임들의 총 가격, 예상 적립금, 결제 버튼이 존재하는 요소입니다.
2. 요소를 position: sticky 설정하여 main 컨텐츠 안에서 fixed처럼 고정될 수 있도록 효과를 줍니다.
3. 예상 적립금은 사용자의 정보를 SELECT하여 사용자의 등급에 따라 적립 퍼센트가 달라질 수 있도록 구현합니다.
4. 적립금은 합계 금액의 등급 퍼센트를 기반으로 적립됩니다.
5. 구매 관련 안내 창을 modal 방식으로 띄웁니다. (관리자 측 공지사항 삭제 기능과 동일한 modal 방식)
6. 체크아웃 버튼을 클릭하면 결제 페이지로 이동하게 됩니다.
게임 플랫폼 설정
1. 해당 게임의 플랫폼 정보가 VARCHAR2 타입으로 저장되어있어 String[]로 뽑아내기 위해 split() 메서드를 사용합니다.
2. select 태그의 기능을 원하는 디자인으로 커스텀하기 위해 기존에 내장되어있는 select 태그가 아닌 div를 이용하여 JS와 CSS로 구현합니다.
3. 해당 게임의 플랫폼이 1개 밖에 없다면 select를 선택할 수 없게끔 로직을 구성합니다.

결제하기 및 결제 완료
- Port One API 사용
- AJAX 사용
- 적립금 사용 및 적립
- 장바구니에 대한 정보 요약
- 우편번호 정규표현식
- 사용자의 Session 세팅
결제 정보 입력
1. 사용자가 체크아웃 버튼을 누르면 controller 요청을 보내 cart 정보를 Model 객체에 싣고 결제 정보를 입력하는 form이 보이는 페이지로 이동합니다.
2. 사용자가 원하는 이름과 주소를 입력하는데, 우편 번호에서 정규표현식을 사용하여 유효성 검사를 진행합니다.
3. 만약, 우편번호 규격에 맞지 않는다면 결제 버튼 기능이 비활성화 됩니다.
4. 이름, 성, 주소, 우편번호를 입력하지 않을 시 동일하게 결제 버튼이 비활성화 되도록 JS에서 처리합니다.
보유한 적립금 사용
1. 사용자가 이전에 구매 이력을 통해 적립된 적립금을 사용할 수 있습니다.
2. 적립금의 전액을 사용하고 싶다면 전액사용 버튼을 누르면 input 태그 값이 변경됩니다.
3. input 태그에는 정규표현식을 이용해 number 타입이 아니라면 입력이 되지 않도록 JS에서 처리합니다.
4. 만약, 사용자가 자신이 보유하고 있는 적립금을 초과하여 값을 입력했다면 input value 값을 읽어 JS에서 비교 후 alert 경고창을 띄우며 보유하고 있는 전액 값으로 변동되게 처리합니다.
5. 전액사용 버튼을 한 번 더 누르면 input 태그 값을 지워 적립금 사용을 취소하도록 합니다.
6. 적립금은 1000원 이상부터 사용이 가능하도록 controller에 담긴 사용자의 보유 적립금을 가져와 1000원 미만이라면 input에 값이 입력되지 않도록 JS에서 처리합니다.
결제 전 장바구니 정보 요약
1. 사용자의 편이성을 위해 장바구니에 담긴 게임의 정보를 요약해서 보여주는 요소입니다.
2. 처음 결제 페이지 진입 시 항목의 개수와 게임 배너 이미지만 보이며, 장바구니 상세 정보 버튼을 클릭하면 상세 요소가 보이도록 JS에서 CSS에서 설정한 class를 추가하는 방식을 사용했습니다.
3. 장바구니에서 선택한 플랫폼 값을 파라미터로 넘겨 받아서 게임 정보에 보여줍니다.
4. 요소가 넘치면 자동으로 스크롤이 생기도록 CSS에서 처리했으며, 스크롤 또한 전체적인 웹 사이트 디자인에 맞춰 설정합니다.
5. 수정 버튼을 누르면 다시 장바구니 페이지로 이동하여 수정할 수 있습니다.
결제 정보 요약
1. 장바구니 페이지에 존재하는 요약 정보와 같은 요소입니다.
2. 요소에 대한 CSS, JSP 코드를 재사용을 하여 작업을 빨리 진행할 수 있도록 했습니다.
Port One API 사용
1. Port One에 가입했던 계정으로 테스트 결제 API를 사용하기 위해 원하는 종류의 결제사를 등록합니다.
2. Port One에서 요구하는 JS 함수와 AJAX POST 타입을 사용하여 결제 정보를 담아 controller로 요청을 보냅니다.
3. 매개변수로 DTO를 지정하여 요청받은 객체를 DB에 INSERT합니다.
4. 결제 창 오른쪽 상단에 상품명과 가격 정보를 보여줍니다. (결제 금액이 너무 큰 관계로 임의로 1원 지정함.)
5. 상품이 2개 이상이라면 "상품명 외 N"으로 간단하게 표시하는 JS 처리를 진행하여 정보를 전달합니다.
결제 완료 페이지
1. AJAX 요청에 응답받은 response 값을 통해 성공적으로 완료가 됐다면, 결제 완료 페이지로 이동합니다.
2. 결제 완료 페이지로 이동할 때 파라미터로 order_id 값을 담아서 넘겨 요청을 받은 controller에서 DB Order 테이블에 해당 PK값을 SELECT하여 Model 객체에 싣습니다.
3. JSP로 이동하기 전 사용자에게 구매한 게임의 키 발급을 위해 game key 테이블에서 해당 게임의 id를 SELECT하고 status가 null인 컬럼을 지정하여 status를 사용자의 ID로 변경합니다.
4. session에 저장된 사용자의 적립금 또한 변경되어야 하기때문에 매개변수로 HttpSession을 통해 기존에 존재하는 사용자의 session을 다시 세팅합니다.
5. 이동한 JSP에서 해당 주문 정보를 사용하여 사용자가 사용한 적립금과 획득한 적립금 정보를 보여주며 총 보유하고 있는 적립금을 보여줍니다.
